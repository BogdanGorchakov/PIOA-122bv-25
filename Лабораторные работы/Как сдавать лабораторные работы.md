# Как сдавать лабораторные работы

## Общая информация

В связи с переходом на GitHub немного меняется процесс сдачи лабораторных работ. Ниже описан актуальный порядок переноса репозитория, отправки работы на проверку и дальнейшего взаимодействия по замечаниям.

## Как перенести репозиторий на GitHub

1. Зарегистрируйтесь на GitHub, если у вас ещё нет аккаунта.
2. Создайте пустой репозиторий на GitHub.
3. В локальной копии вашего проекта выполните следующие команды:

```bash
# Удаляем старую привязку к репозиторию в GitLab.
git remote remove origin

# Добавляем новый удалённый репозиторий на GitHub.
# Вместо <URL_ВАШЕГО_РЕПОЗИТОРИЯ> нужно указать ссылку в формате HTTPS или SSH.
git remote add origin <URL_ВАШЕГО_РЕПОЗИТОРИЯ>

# Отправляем все ветки в новый репозиторий.
git push -u origin --all
```

После этого ваш проект будет доступен на GitHub.

## Как отправить лабораторную на проверку

После того как работа будет готова, порядок сдачи следующий:

1. Создайте `Pull Request`.
   Это аналог `Merge Request` из GitLab.
2. Скопируйте ссылку на ваш `Pull Request`.
3. Отправьте письмо на почту `konoplevte@mai.ru`.
4. В письме обязательно укажите:
   - ссылку на `Pull Request`
   - ваше ФИО
   - номер вашей группы

## Доступ к репозиторию

В некоторых случаях может понадобиться дополнительно выдать доступ к репозиторию для проверки.

Если это потребуется, добавьте пользователя:

`https://github.com/konoplevte`

в качестве участника вашего репозитория.

## Как проходит проверка

Проверка лабораторных будет происходить через `Pull Request`, так же как раньше это делалось через `Merge Request` в GitLab.

Замечания и комментарии будут оставляться прямо в вашем `Pull Request`.

## Важное правило по повторной отправке

Если вы:

- переотправляете работу,
- вносите исправления после замечаний,
- создаёте новый `Pull Request`,

то это обязательно должно сопровождаться новым письмом на почту.

Это нужно для того, чтобы сохранялась понятная история взаимодействия по вашей работе.

Если письма нет, считается, что работа на повторную проверку не отправлена.
