### Цели

1. Освоить базовые приёмы работы с файлами в Python.
2. Научиться сохранять и загружать данные приложения между запусками программы.
3. Закрепить применение объектно-ориентированного подхода при расширении существующей архитектуры.
4. Получить практический опыт реализации файловой СУБД с единым интерфейсом доступа к данным.

### Основная задача (3 балла)

Задание необходимо выполнять в ветке `task4` вашего основного репозитория `PIOA НомерГруппы`.

Необходимо расширить систему управления данными, реализованную во второй и третьей лабораторных работах, добавив поддержку файловой СУБД.

Требования к базовому решению:

1. Сохранить объектно-ориентированную архитектуру, полученную в предыдущей лабораторной работе.
2. Реализовать отдельный класс файловой базы данных, работающий через тот же интерфейс, что и `in-memory` реализация.
3. Обеспечить хранение данных на диске, чтобы после завершения программы записи не терялись.
4. Поддержать операции, реализованные в основной части предыдущих лабораторных работ.
5. Реализовать хранение структуры таблицы и её записей в файлах в формате `JSON`.
6. Обеспечить обработку ошибок, связанных с работой с файлами и некорректными пользовательскими данными.
7. Актуализировать автоматизированные тесты для новой функциональности.
8. Обновить файл `README.md`, отразив:
   - структуру проекта;
   - различия между `in-memory` и файловой реализациями;
   - инструкцию по запуску и проверке программы.
9. Соблюдать принципы чистого кода и общепринятые стандарты оформления кода.

### Дополнительная задача (4 балла)

Помимо базовой части, необходимо добавить реализацию файловой СУБД в другом текстовом формате хранения данных, например `CSV`.

При выполнении задания требуется:

1. Сохранить совместимость публичного интерфейса между различными файловыми реализациями.
2. Реализовать отдельный класс файловой базы данных для нового формата хранения.
3. Обеспечить корректное сохранение и загрузку структуры таблицы и её записей в выбранном формате.
4. Сохранить поддержку операций, реализованных в базовой части лабораторной работы.
5. Добавить тесты для новой реализации.

### Дополнительная задача (5 баллов)

Помимо предыдущих требований, необходимо добавить возможность индексации данных для ускорения работы методов СУБД.

При выполнении задания требуется:

1. Реализовать механизм индексации по одному или нескольким полям таблицы.
2. Обеспечить использование индексов в методах чтения данных.
3. Сохранить корректность работы СУБД при добавлении новых записей.
4. Если в проекте реализованы операции изменения и удаления записей, обеспечить корректное обновление индексов после таких операций.
5. Добавить тесты для сценариев с индексами.
