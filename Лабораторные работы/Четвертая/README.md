# Лабораторная работа №4 — Файловая СУБД (JSON / CSV) с сохранением состояния

### 1. Структура проекта
* **Лабораторные работы / Четвертая / main.py** — основной скрипт СУБД с реализацией сохранения на диск и юнит-тестами.
* **Лабораторные работы / Четвертая / README.md** — текущий файл документации.

### 2. Различия между In-Memory и Файловой реализациями
* **In-Memory СУБД (Лаб №2, №3)**: Хранит все данные исключительно в оперативной памяти компьютера. При закрытии программы или выключении ноутбука все добавленные записи безвозвратно теряются.
* **Файловая СУБД (Лаб №4)**: При любом добавлении или удалении записи автоматически синхронизирует состояние с жестким диском. При повторном запуске программы данные считываются из файлов, обеспечивая постоянное хранение.

### 3. Реализованная функциональность
* **JSON Хранение (Основная задача)**: Реализован класс `JsonDatabase`, сериализующий объекты в структурированный текстовый формат JSON.
* **CSV Хранение (Дополнительная задача +4 балла)**: Спроектирован и реализован альтернативный класс `CsvDatabase` для работы с табличными данными в формате CSV через встроенный модуль `csv`.
* Общий публичный интерфейс сохранен для обоих файловых классов.
* Написаны автоматические тесты файловых сценариев чтения/записи.

### 4. Инструкция по запуску
1. Запустите скрипт командой: `python main.py`
2. Программа выполнит тесты, после чего предложит выбрать формат файла (1 - JSON, 2 - CSV) и откроет меню.
