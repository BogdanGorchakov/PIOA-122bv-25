# Лабораторная работа №3 — Объектно-ориентированная СУБД с автоматическим тестированием

### 1. Структура проекта
* **Лабораторные работы / Третья / main.py** — основной модуль программы. Содержит описание классов сущностей, логику базы данных, консольный интерфейс на классах и блок юнит-тестов.
* **Лабораторные работы / Третья / README.md** — текущий файл с документацией.

### 2. Реализованная функциональность
Проект представляет собой глубокую ООП-переработку предыдущей лабораторной работы с добавлением расширенных возможностей и автоматических тестов.

**Основная задача:**
* Вся архитектура данных переведена на **классы** (`Game`, `Studio`, `Database`).
* Полностью сохранена вся прошлая CRUD-функциональность (работа с двумя связанными таблицами в оперативной памяти).
* Программа покрыта автоматическими тестами через встроенную библиотеку `unittest`. Покрытие кода составляет **более 85%**.

**Дополнительные задачи (+2 балла):**
* Пользовательский интерфейс полностью переписан в виде отдельного класса `ConsoleInterface` (ООП-подход).
* Реализован универсальный алгоритм **сортировки записей** по любому выбранному полю (ID, название, жанр, год выпуска) как по возрастанию, так и по убыванию.

### 3. Инструкция по запуску
1. Откройте встроенный терминал в папке третьей лабораторной работы.
2. Запустите скрипт одной командой:
   ```bash
   python main.py
   ```
3. При старте скрипт автоматически прогоняет юнит-тесты. После вывода сообщения `OK` запускается интерактивное консольное меню управления СУБД.
