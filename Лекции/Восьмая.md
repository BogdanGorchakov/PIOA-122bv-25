# Работа с файлами

На последней лабораторной работе вам предстоит работать с файлами. Давайте подробнее разберём, как Python работает с файлами, что происходит при чтении и записи текста, зачем нужны кодировки и какие форматы хранения данных используются чаще всего.

## Кодировки

Прежде чем говорить про файлы, нужно обсудить строки и кодировки. Современные программы очень часто работают с текстом: читают его из файлов, получают по сети, выводят на экран, сохраняют в базу данных. Поэтому важно понимать, как текст вообще хранится в памяти компьютера.

Компьютер не хранит буквы "как буквы". Он хранит числа, а числа уже интерпретируются как символы по определённым правилам. Эти правила и называются кодировкой.

Если говорить очень грубо, то любая работа с текстом сводится к двум операциям:

1. Преобразовать строку в байты.
2. Преобразовать байты обратно в строку.

В Python эти операции выглядят так:

```python
text = "hello"

# encode() превращает строку в байты.
data = text.encode("utf-8")
print(data)  # b'hello'

# decode() превращает байты обратно в строку.
decoded_text = data.decode("utf-8")
print(decoded_text)  # hello
```

Если кодировка выбрана неправильно, то текст либо испортится, либо Python выдаст ошибку.

## ASCII

Основное развитие компьютерных технологий долгое время происходило в США, поэтому неудивительно, что одна из первых популярных кодировок была ориентирована именно на английский язык. Эта кодировка называется `ASCII`.

Идея ASCII очень простая:

- составляется таблица из `128` символов
- каждому символу назначается номер от `0` до `127`
- такого диапазона хватает на латинские буквы, цифры, знаки препинания и служебные символы

Число `128` помещается в `7` бит, но на практике символ всё равно обычно хранится в одном байте, то есть в `8` битах.

Например:

- `o` имеет код `111`
- `k` имеет код `107`

Значит, строка `"ok"` в ASCII представляется как два байта:

- `01101111`
- `01101011`

Процесс декодирования выполняется в обратную сторону: по числу восстанавливается символ.

Покажем это на примере Python.

```python
text = "ok"

# Преобразуем строку в байты в ASCII-кодировке.
ascii_bytes = text.encode("ascii")
print(ascii_bytes)  # b'ok'

# Каждый элемент объекта bytes - это число от 0 до 255.
# Для строки "ok" это будут коды символов 'o' и 'k'.
for byte in ascii_bytes:
    print(byte)
    # Первая строка вывода: 111
    # Вторая строка вывода: 107

# Покажем эти же байты в двоичном виде.
for byte in ascii_bytes:
    print(bin(byte))
    # Первая строка вывода: 0b1101111
    # Вторая строка вывода: 0b1101011
    # Ведущие нули Python не показывает, потому что они не влияют на значение.

# Кириллицу в ASCII закодировать нельзя,
# потому что русских букв в этой таблице просто нет.
"привет".encode("ascii")  # UnicodeEncodeError
```

ASCII была отличным решением для английского текста, но у неё есть фундаментальное ограничение: `128` символов слишком мало для всех языков мира.

## Однобайтные национальные кодировки

Когда стало понятно, что ASCII недостаточно, начали появляться так называемые однобайтные национальные кодировки. Идея у них была примерно следующая:

- первые `128` кодов оставить совместимыми с ASCII
- вторую половину таблицы, то есть диапазон `128-255`, использовать под символы конкретного языка

Для кириллицы одной из самых известных таких кодировок стала `Windows-1251`, в Python она обычно обозначается как `cp1251`.

```python
text = "ок"

# Кодируем строку в байты в кодировке CP1251.
cp1251_bytes = text.encode("cp1251")
print(cp1251_bytes)  # b'\xee\xea'

# В repr-форме Python показывает не сами буквы,
# а шестнадцатеричное представление байтов.
# \xee - это байт для 'о'
# \xea - это байт для 'к'

for byte in cp1251_bytes:
    print(byte)
    # Первая строка вывода: 238
    # Вторая строка вывода: 234

for byte in cp1251_bytes:
    print(bin(byte))
    # Первая строка вывода: 0b11101110
    # Вторая строка вывода: 0b11101010

# Символы английского алфавита в CP1251 совпадают с ASCII.
latin_bytes = "hello".encode("cp1251")
print(latin_bytes)  # b'hello'
```

У такого подхода были очевидные проблемы:

1. Для каждого языка приходилось создавать свою кодировку.
2. В один байт помещается только `256` значений, а этого мало для всех письменностей мира.
3. Если файл был записан в одной локальной кодировке, а прочитан в другой, текст превращался в "кракозябры".

Именно поэтому со временем появился `Unicode`.

## Unicode и UTF-8

Здесь важно не путать две разные идеи.

`Unicode` - это не одна конкретная кодировка. Это общая таблица символов, в которой каждому символу соответствует свой кодовый номер.

Например:

- `A` - это `U+0041`
- `о` - это `U+043E`
- `к` - это `U+043A`

Запись вида `U+043E` называется кодовой позицией символа в Unicode.

Но сам по себе Unicode ещё не отвечает на вопрос, как именно хранить этот символ в байтах. Для этого существуют конкретные кодировки Unicode, например:

- `UTF-8`
- `UTF-16`
- `UTF-32`

В учебной и практической разработке чаще всего встречается именно `UTF-8`.

### Идея UTF-8

`UTF-8` кодирует символы в разное количество байт:

- символы ASCII кодируются в `1` байт
- многие символы кириллицы кодируются в `2` байта
- многие символы азиатских письменностей кодируются в `3` байта
- некоторые редкие символы и эмодзи кодируются в `4` байта

Главное достоинство UTF-8 в том, что она обратно совместима с ASCII. То есть любой ASCII-текст уже является корректным UTF-8-текстом.

### Как UTF-8 понимает длину символа

У UTF-8 есть специальная схема, по которой можно понять, сколько байт занимает символ.

Если смотреть на старшие биты первого байта, то правило такое:

- если первый бит равен `0`, значит символ занимает `1` байт
- если первые три бита равны `110`, значит символ занимает `2` байта
- если первые четыре бита равны `1110`, значит символ занимает `3` байта
- если первые пять бит равны `11110`, значит символ занимает `4` байта

Для всех дополнительных байтов действует другое правило:

- байт-продолжение начинается с битов `10`

Именно за счёт этой схемы UTF-8 может однозначно отделять начало символа от его продолжения.

Можно запомнить это в виде таблицы:

| Количество байт | Вид первого байта | Вид остальных байтов |
| --------------- | ----------------- | -------------------- |
| 1               | `0xxxxxxx`        | нет                  |
| 2               | `110xxxxx`        | `10xxxxxx`           |
| 3               | `1110xxxx`        | `10xxxxxx`           |
| 4               | `11110xxx`        | `10xxxxxx`           |

### Пример UTF-8 для кириллицы

Посмотрим, как кодируется строка `"ок"`.

```python
text = "ок"

# Кодируем строку в UTF-8.
utf8_bytes = text.encode("utf-8")
print(utf8_bytes)  # b'\xd0\xbe\xd0\xba'

# Здесь каждый русский символ занимает два байта.
# Символ 'о' -> \xd0\xbe
# Символ 'к' -> \xd0\xba

for byte in utf8_bytes:
    print(byte)
    # Последовательно будут выведены:
    # 208
    # 190
    # 208
    # 186

for byte in utf8_bytes:
    print(bin(byte))
    # Последовательно будут выведены:
    # 0b11010000
    # 0b10111110
    # 0b11010000
    # 0b10111010
```

Если внимательно посмотреть на двоичную форму, можно заметить:

- первый байт символа начинается с `110`, то есть это начало двухбайтного символа
- второй байт начинается с `10`, то есть это продолжение того же символа

Именно так UTF-8 и понимает структуру многобайтного текста.

### Почему UTF-8 так популярен

У UTF-8 есть несколько важных достоинств:

1. Полная совместимость с ASCII.
2. Экономное хранение английского текста.
3. Универсальность: можно представить символы практически всех письменностей мира.
4. Удобство для обмена данными по сети и хранения текстовых файлов.

Именно поэтому UTF-8 сегодня является основным стандартом для веба, API, JSON-файлов, исходного кода и большинства современных текстовых систем.

### А что с UTF-16 и UTF-32

Есть и другие способы представления Unicode.

`UTF-16` обычно использует `2` или `4` байта на символ. `UTF-32` использует фиксированные `4` байта на символ.

Если очень грубо упростить:

- `UTF-8` обычно экономит память на текстах, где много ASCII-символов
- `UTF-16` и `UTF-32` могут быть удобнее для некоторых внутренних операций обработки текста
- за удобство обработки часто приходится платить большим расходом памяти

Для передачи данных и хранения текстовых файлов почти всегда предпочтителен именно `UTF-8`.

### Итог по кодировкам

Я рассказываю всё это для того, чтобы вы понимали: когда вы работаете со строками и файлами, не происходит никакой магии. Есть таблицы символов, есть правила кодирования, есть байты, есть операции декодирования. Всё это придумано и стандартизовано, чтобы программы могли одинаково понимать текст.

## Работа с файлами

Теперь перейдём к самим файлам.

Файл для программы - это источник или приёмник данных. Из файла можно читать, в файл можно писать. Чаще всего нас интересуют два типа файлов:

- текстовые
- бинарные

С текстовыми файлами мы работаем как со строками. С бинарными - как с байтами.

## Открытие файла

Чтобы открыть файл, используется функция `open()`.

```python
# Первый аргумент - путь к файлу.
# Второй аргумент - режим открытия.
# encoding указывается для текстовых файлов.
file = open("file.txt", "r", encoding="utf-8")

# После завершения работы файл нужно закрыть.
file.close()
```

У `open()` есть несколько часто используемых режимов.

```python
# Чтение текстового файла.
# Если файла не существует, будет ошибка FileNotFoundError.
file = open("file.txt", "r", encoding="utf-8")
file.close()

# Запись в файл.
# Если файл уже существует, его старое содержимое будет удалено.
file = open("file.txt", "w", encoding="utf-8")
file.close()

# Добавление в конец файла.
# Если файла нет, он будет создан.
file = open("file.txt", "a", encoding="utf-8")
file.close()

# Создание нового файла.
# Если файл уже существует, будет ошибка FileExistsError.
file = open("file.txt", "x", encoding="utf-8")
file.close()

# Открытие бинарного файла для чтения.
file = open("image.png", "rb")
file.close()
```

Стоит отдельно заметить:

- `b` означает бинарный режим
- `t` означает текстовый режим

Текстовый режим используется по умолчанию, поэтому `"r"` и `"rt"` эквивалентны.

## Почему файлы нужно закрывать

После открытия файл связан с ресурсами операционной системы. Если не закрывать файлы, можно получить утечки ресурсов, проблемы с буферами и другие неприятные эффекты.

```python
file = open("file.txt", "r", encoding="utf-8")
content = file.read()
print(content)  # Например: Привет, мир!
file.close()
```

Но вручную закрывать файл неудобно и небезопасно. Если до `close()` произойдёт ошибка, файл может так и остаться открытым.

Именно поэтому в Python обычно используют `with`.

## Конструкция with

Конструкция `with` автоматически закрывает файл после завершения блока кода.

```python
with open("file.txt", "r", encoding="utf-8") as file:
    content = file.read()
    print(content)
    # Если в file.txt лежит:
    # Привет
    # Мир
    # то вывод будет:
    # Привет
    # Мир
```

Это рекомендуемый способ работы с файлами.

## Чтение файла

У файлового объекта есть несколько основных методов чтения.

### read()

Метод `read()` читает всё содержимое файла целиком.

```python
with open("file.txt", "r", encoding="utf-8") as file:
    content = file.read()
    print(content)
    # Если в файле:
    # Привет
    # Мир
    # то вывод будет:
    # Привет
    # Мир
```

Это удобно для маленьких файлов. Но для больших файлов такой подход может быть опасен, потому что всё содержимое загружается в память сразу.

### readlines()

Метод `readlines()` читает файл в список строк.

```python
with open("file.txt", "r", encoding="utf-8") as file:
    lines = file.readlines()
    print(lines)
    # Если в файле:
    # Привет
    # Мир
    # то вывод будет:
    # ['Привет\n', 'Мир']
```

Обратите внимание: символы перевода строки обычно сохраняются.

### readline()

Метод `readline()` читает только одну строку.

```python
with open("file.txt", "r", encoding="utf-8") as file:
    line = file.readline()
    print(line)
    # Если первая строка файла "Привет\n",
    # то вывод будет:
    # Привет
```

Если вызвать `readline()` несколько раз, строки будут читаться последовательно.

```python
with open("file.txt", "r", encoding="utf-8") as file:
    print(file.readline())  # Первая строка файла, например: Привет
    print(file.readline())  # Вторая строка файла, например: Мир
```

### Чтение в цикле

Очень часто файл удобнее читать построчно.

```python
with open("file.txt", "r", encoding="utf-8") as file:
    for line in file:
        print(line.strip())
        # Если в файле строки:
        # Привет
        # Мир
        # то вывод будет:
        # Привет
        # Мир
```

Именно этот способ обычно лучше всего подходит для больших текстовых файлов.

## Запись в файл

### write()

Метод `write()` записывает строку в файл.

```python
with open("file.txt", "w", encoding="utf-8") as file:
    written_count_1 = file.write("Привет мир!\n")
    written_count_2 = file.write("Это новая строка.\n")

    print(written_count_1)  # 12
    print(written_count_2)  # 18
```

Метод `write()` возвращает количество записанных символов.

После выполнения этого кода содержимое файла станет таким:

```text
Привет мир!
Это новая строка.
```

### Добавление в конец файла

```python
with open("file.txt", "a", encoding="utf-8") as file:
    written_count = file.write("Эта строка будет добавлена в конец файла.\n")
    print(written_count)  # 42
```

После выполнения этой операции строка окажется в конце файла.

### writelines()

`writelines()` записывает последовательность строк.

```python
lines = ["Строка 1\n", "Строка 2\n", "Строка 3\n"]

with open("file.txt", "w", encoding="utf-8") as file:
    file.writelines(lines)

with open("file.txt", "r", encoding="utf-8") as file:
    print(file.read())
    # Вывод будет:
    # Строка 1
    # Строка 2
    # Строка 3
```

Важно: `writelines()` не добавляет `\n` автоматически. Если переводы строк нужны, их надо включить в сами строки.

## Указатель файла

У файла есть текущая позиция чтения или записи. Иногда это называют указателем или курсором файла.

Для работы с ним используются методы:

- `tell()` - узнать текущую позицию
- `seek()` - перейти в другую позицию

```python
with open("file.txt", "r", encoding="utf-8") as file:
    position = file.tell()
    print(position)  # 0

    content = file.read(5)
    print(content)  # Например: Приве

    position = file.tell()
    print(position)  # 5

    file.seek(0)
    print(file.tell())  # 0
```

Это полезно, когда нужно перечитать файл с начала или перейти к определённому месту.

## Что будет при повторном read()

После чтения файла указатель смещается вперёд. Если файл уже прочитан до конца, следующий вызов `read()` вернёт пустую строку.

```python
with open("file.txt", "r", encoding="utf-8") as file:
    first_read = file.read()
    print(first_read)
    # Например:
    # Привет
    # Мир

    second_read = file.read()
    print(second_read)  # Пустая строка
```

Если нужно прочитать файл ещё раз, указатель надо вернуть в начало.

```python
with open("file.txt", "r", encoding="utf-8") as file:
    print(file.read())
    # Например:
    # Привет
    # Мир

    file.seek(0)

    print(file.read())
    # Вывод снова будет:
    # Привет
    # Мир
```

## Обработка исключений

При работе с файлами очень часто возникают ошибки внешней среды:

- файла не существует
- у программы нет прав доступа
- путь указан неверно

Поэтому полезно использовать `try/except`.

```python
try:
    with open("non_existent_file.txt", "r", encoding="utf-8") as file:
        content = file.read()
except FileNotFoundError:
    print("Файл не существует.")
except PermissionError:
    print("У вас нет прав на чтение файла.")
```

## Работа с бинарными файлами

Текстовые файлы состоят из текста, который декодируется в строки. Но изображения, архивы, аудиофайлы и видеофайлы устроены иначе. Их читают в бинарном режиме.

```python
with open("image.png", "rb") as file:
    data = file.read()
    print(type(data))  # <class 'bytes'>
```

В бинарном режиме мы получаем объект `bytes`, а не строку.

## Что происходит при open()

Давайте теперь чуть подробнее рассмотрим, что происходит при вызове `open()`. Это полезно, чтобы понимать, что работа с файлами - это обычный ввод-вывод через операционную систему, а не какая-то магия.

Когда мы вызываем:

```python
with open("file.txt", "r", encoding="utf-8") as file:
    content = file.read()
```

происходит примерно следующее.

1. Python проверяет параметры:
   - корректен ли путь
   - корректен ли режим
   - нужно ли открыть файл в текстовом или бинарном виде
   - какая кодировка должна использоваться
   - нужна ли буферизация
2. Python переходит к низкоуровневому слою ввода-вывода.
3. Выполняется системный вызов открытия файла.
4. Операционная система ищет файл по указанному пути.
5. ОС проверяет:
   - существует ли такой путь
   - действительно ли там файл
   - есть ли у процесса нужные права
   - можно ли открыть файл в заданном режиме
6. ОС создаёт внутреннюю запись об открытом файле и выдаёт процессу файловый дескриптор.
7. Python создаёт объект низкого уровня, обычно связанный с `io.FileIO`.
8. Поверх него создаётся буферизированный слой, например `io.BufferedReader` или `io.BufferedWriter`.
9. Если файл открыт в текстовом режиме, поверх буфера создаётся текстовая обёртка `io.TextIOWrapper`, которая умеет декодировать байты в строки и кодировать строки обратно в байты.
10. В итоге функция `open()` возвращает объект файла, с которым мы уже работаем в Python-коде.

## Зачем нужна буферизация

Операции ввода-вывода заметно медленнее операций с памятью. Если бы программа обращалась к диску за каждым символом или байтом отдельно, всё работало бы очень медленно.

Поэтому данные обычно читаются и записываются не по одному байту, а блоками. Эти блоки временно хранятся в памяти. Такой промежуточный участок памяти и называется буфером.

Идея буферизации проста:

- уменьшить количество обращений к операционной системе
- читать и писать более крупными порциями
- ускорить работу программы

## Что происходит при read()

При вызове `read()` происходит примерно следующее:

1. Python обращается к файловому объекту.
2. Буфер проверяет, есть ли нужные данные уже в памяти.
3. Если буфер пуст или данных недостаточно, происходит системный вызов к ОС.
4. ОС читает данные из файла по файловому дескриптору.
5. Полученные байты попадают в буфер.
6. Если файл текстовый, Python декодирует байты в строку по выбранной кодировке.

Именно поэтому `read()` на текстовом файле возвращает строку, а не байты.

## Что происходит при write()

При вызове `write()` процесс идёт в обратную сторону:

1. Python берёт строку.
2. Кодирует её в байты по выбранной кодировке.
3. Помещает байты в буфер.
4. Буфер позже записывает их в файл.

Фактическая запись на диск обычно происходит, когда:

- буфер переполняется
- вызывается `flush()`
- вызывается `close()`

## Почему read() может быть опасен

`read()` без аргументов читает весь файл целиком. Это удобно, когда файл маленький. Но если файл очень большой, программа может потратить слишком много памяти.

Поэтому в реальной разработке часто лучше:

- читать файл построчно
- читать фиксированными кусками
- обрабатывать данные постепенно

## Работа с путями через os

При работе с файлами полезно уметь собирать и анализировать пути. Один из классических способов - использовать модуль `os`.

```python
import os

# Получаем текущую рабочую директорию.
current_directory = os.getcwd()
print(current_directory)  # Например: /home/user/project

# Формируем путь к файлу.
file_path = os.path.join(current_directory, "file.txt")
print(file_path)  # Например: /home/user/project/file.txt

# Проверяем существование пути.
if os.path.exists(file_path):
    print("Файл существует.")

# Получаем имя файла.
file_name = os.path.basename(file_path)
print(file_name)  # file.txt

# Получаем директорию.
directory = os.path.dirname(file_path)
print(directory)  # Например: /home/user/project
```

## Работа с путями через pathlib

В современном Python часто удобнее использовать `pathlib`.

```python
from pathlib import Path

path = Path("file.txt")

if path.exists():
    print("Есть файл") 

# Читаем текст из файла.
content = path.read_text(encoding="utf-8")
print(content)
# Если в файле лежит Hello,
# то вывод будет:
# Hello

# Записываем новый текст в файл.
path.write_text("Hello", encoding="utf-8")

# Снова читаем файл после записи.
content = path.read_text(encoding="utf-8")
print(content)  # Hello
```

`pathlib` особенно хорош тем, что делает код более читаемым и удобным.

## Файловые форматы

Файл сам по себе - это просто последовательность байтов. Но чтобы разные программы могли одинаково понимать структуру данных, используются форматы хранения информации.

Рассмотрим три популярных формата:

- `CSV`
- `JSON`
- `XML`

## CSV

`CSV` (`Comma-Separated Values`) - это простой табличный текстовый формат. Обычно:

- в первой строке находятся заголовки столбцов
- в остальных строках - сами данные
- значения разделяются специальным символом, чаще всего запятой или точкой с запятой

Пример CSV:

```csv
student_id,first_name,second_name,address
1,Ivan,Ivanov,"Moscow, Red Square 1"
2,Irina,Petrova,"Moscow, Red Square 2"
```

Если внутри значения есть запятая, значение обычно заключают в кавычки.

### Чтение CSV

```python
import csv

with open("file.csv", mode="r", encoding="utf-8", newline="") as file:
    reader = csv.reader(file)
    for row in reader:
        print(row)
        # Первая строка вывода:
        # ['student_id', 'first_name', 'second_name', 'address']
        # Вторая строка вывода:
        # ['1', 'Ivan', 'Ivanov', 'Moscow, Red Square 1']
        # Третья строка вывода:
        # ['2', 'Irina', 'Petrova', 'Moscow, Red Square 2']
```

### Чтение CSV как словаря

```python
import csv

with open("file.csv", mode="r", encoding="utf-8", newline="") as file:
    reader = csv.DictReader(file)
    for row in reader:
        print(row)
        # Первая строка вывода:
        # {'student_id': '1', 'first_name': 'Ivan', 'second_name': 'Ivanov', 'address': 'Moscow, Red Square 1'}
        # Вторая строка вывода:
        # {'student_id': '2', 'first_name': 'Irina', 'second_name': 'Petrova', 'address': 'Moscow, Red Square 2'}
```

### Запись в CSV

```python
import csv

with open("file.csv", mode="w", encoding="utf-8", newline="") as file:
    writer = csv.writer(file)
    writer.writerow(["student_id", "first_name", "second_name", "address"])
    writer.writerow(["1", "Ivan", "Ivanov", "Moscow, Red Square 1"])
    writer.writerow(["2", "Irina", "Petrova", "Moscow, Red Square 2"])
```

После записи содержимое файла будет таким:

```csv
student_id,first_name,second_name,address
1,Ivan,Ivanov,Moscow, Red Square 1
2,Irina,Petrova,Moscow, Red Square 2
```

Если нужно, модуль `csv` сам поставит кавычки там, где это требуется.

### Запись в CSV через словарь

```python
import csv

with open("file.csv", mode="w", encoding="utf-8", newline="") as file:
    fieldnames = ["student_id", "first_name", "second_name", "address"]
    writer = csv.DictWriter(file, fieldnames=fieldnames)
    writer.writeheader()
    writer.writerow({
        "student_id": "1",
        "first_name": "Ivan",
        "second_name": "Ivanov",
        "address": "Moscow, Red Square 1",
    })
    writer.writerow({
        "student_id": "2",
        "first_name": "Irina",
        "second_name": "Petrova",
        "address": "Moscow, Red Square 2",
    })
```

CSV часто используется в:

- Excel и других табличных редакторах
- простом импорте и экспорте данных
- отчётах
- логах

## JSON

`JSON` (`JavaScript Object Notation`) - это текстовый формат представления структурированных данных. По сути он очень похож на словари и списки, поэтому в Python с ним особенно удобно работать.

Пример JSON:

```json
{
    "data": [
        {
            "student_id": 1,
            "first_name": "Ivan",
            "second_name": "Ivanov",
            "address": "Moscow, Red Square 1"
        },
        {
            "student_id": 2,
            "first_name": "Irina",
            "second_name": "Petrova",
            "address": "Moscow, Red Square 2"
        }
    ]
}
```

У JSON есть базовые типы данных:

1. число: `1`, `1.24`, `-2`
2. строка: `"тест"`
3. объект: `{"key": "value"}`
4. массив: `[1, 2, 3]`
5. булевы значения: `true`, `false`
6. пустое значение: `null`

Важно помнить, что ключи в JSON всегда задаются строками.

### Чтение JSON

```python
import json

with open("file.json", mode="r", encoding="utf-8") as file:
    data = json.load(file)
    print(data)
    # Вывод будет:
    # {'data': [{'student_id': 1, 'first_name': 'Ivan', 'second_name': 'Ivanov', 'address': 'Moscow, Red Square 1'},
    #           {'student_id': 2, 'first_name': 'Irina', 'second_name': 'Petrova', 'address': 'Moscow, Red Square 2'}]}
```

### Запись JSON

```python
import json

data = {
    "data": [
        {
            "student_id": 1,
            "first_name": "Ivan",
            "second_name": "Ivanov",
            "address": "Moscow, Red Square 1",
        },
        {
            "student_id": 2,
            "first_name": "Ирина",
            "second_name": "Петрова",
            "address": "Moscow, Red Square 2",
        },
    ]
}

with open("file.json", mode="w", encoding="utf-8") as file:
    json.dump(data, file, ensure_ascii=False, indent=4)
```

После записи файл будет выглядеть так:

```json
{
    "data": [
        {
            "student_id": 1,
            "first_name": "Ivan",
            "second_name": "Ivanov",
            "address": "Moscow, Red Square 1"
        },
        {
            "student_id": 2,
            "first_name": "Ирина",
            "second_name": "Петрова",
            "address": "Moscow, Red Square 2"
        }
    ]
}
```

Параметр `ensure_ascii=False` нужен для того, чтобы кириллица сохранялась в читаемом виде, а не в виде escape-последовательностей.

JSON особенно часто используется:

- в веб-разработке
- в API
- в конфигурационных файлах
- при обмене данными между сервисами

## XML

`XML` (`eXtensible Markup Language`) - это ещё один текстовый формат хранения структурированных данных.

На текущий момент он уже не так популярен, как JSON, но по-прежнему встречается в старых системах, корпоративных интеграциях и некоторых специализированных форматах документов.

Пример XML:

```xml
<data>
    <student>
        <student_id>1</student_id>
        <first_name>Ivan</first_name>
        <second_name>Ivanov</second_name>
        <address>Moscow, Red Square 1</address>
    </student>
    <student>
        <student_id>2</student_id>
        <first_name>Irina</first_name>
        <second_name>Petrova</second_name>
        <address>Moscow, Red Square 2</address>
    </student>
</data>
```

По сравнению с JSON XML более многословен и менее удобен для ручного чтения, но его структура очень формальна и расширяема.

### Чтение XML

```python
import xml.etree.ElementTree as ET

tree = ET.parse("file.xml")
root = tree.getroot()

print(f"Корневой элемент: {root.tag}")  # Корневой элемент: data

for student in root.findall("student"):
    student_id = student.findtext("student_id")
    first_name = student.findtext("first_name")
    second_name = student.findtext("second_name")
    address = student.findtext("address")

    print(f"id: {student_id}")
    print(f"Имя: {first_name}")
    print(f"Фамилия: {second_name}")
    print(f"Адрес: {address}")
    print()
    # Для первого student вывод будет:
    # id: 1
    # Имя: Ivan
    # Фамилия: Ivanov
    # Адрес: Moscow, Red Square 1
    #
    # Для второго student вывод будет:
    # id: 2
    # Имя: Irina
    # Фамилия: Petrova
    # Адрес: Moscow, Red Square 2
```

### Запись XML

```python
import xml.etree.ElementTree as ET

students = ET.Element("students")

student_1 = ET.SubElement(students, "student")
ET.SubElement(student_1, "student_id").text = "1"
ET.SubElement(student_1, "first_name").text = "Ivan"
ET.SubElement(student_1, "second_name").text = "Ivanov"
ET.SubElement(student_1, "address").text = "Moscow, Red Square 1"

student_2 = ET.SubElement(students, "student")
ET.SubElement(student_2, "student_id").text = "2"
ET.SubElement(student_2, "first_name").text = "Irina"
ET.SubElement(student_2, "second_name").text = "Petrova"
ET.SubElement(student_2, "address").text = "Moscow, Red Square 2"

tree = ET.ElementTree(students)
ET.indent(tree, space="    ")
tree.write("file.xml", encoding="utf-8")
```

После записи файл будет выглядеть примерно так:

```xml
<students>
    <student>
        <student_id>1</student_id>
        <first_name>Ivan</first_name>
        <second_name>Ivanov</second_name>
        <address>Moscow, Red Square 1</address>
    </student>
    <student>
        <student_id>2</student_id>
        <first_name>Irina</first_name>
        <second_name>Petrova</second_name>
        <address>Moscow, Red Square 2</address>
    </student>
</students>
```

## Сравнение форматов

Если сильно упростить:

- `CSV` хорош для простых таблиц
- `JSON` хорош для структурированных данных и вложенных объектов
- `XML` используется там, где нужен более формальный и расширяемый формат или где он исторически уже принят

У каждого формата есть свои сильные и слабые стороны. Важно не искать "лучший" формат вообще, а выбирать формат под конкретную задачу.
