## Вводная часть

На предыдущей лекции при обсуждении принципов объектно-ориентированного программирования (ООП) стало очевидно, что язык Python не является типичным представителем строго объектно-ориентированных языков. Вследствие этого часть примеров может выглядеть для студентов не вполне очевидной.

В данной лекции предлагается повторить основные постулаты ООП, опираясь на язык C# как пример классического объектно-ориентированного языка и на язык Go как пример языка с нетипичной реализацией объектно-ориентированных идей. В завершение лекции анализ будет вновь возвращен к Python, чтобы сопоставить подходы и подчеркнуть особенности языка.

## Инкапсуляция

Рассмотрим принцип инкапсуляции на примере простой геометрической сущности — прямоугольника. Сначала реализуем данный пример на Python в файле `shapes.py`.

```python
# Класс прямоугольник
class Rectangle:
    def __init__(self, width, height):
        # У прямоугольника два "приватных" поля: ширина и высота
        self._width = width
        self._height = height

    # Метод вычисления площади прямоугольника
    def area(self):
        return self._width * self._height

    # Метод вычисления периметра прямоугольника
    def perimeter(self):
        return 2 * (self._width + self._height)
```

Далее запишем аналогичный класс на C# в файле `shapes.cs`.

```csharp
// Задаем пространство имен для нашей программы
namespace Shapes;

// Объявляем публичный класс "прямоугольник". Следует отметить, что классы
// также инкапсулированы внутри пространства имен.
public class Rectangle
{
    // Два приватных поля: ширина и высота
    // Для приватных полей используется ключевое слово private
    private double _width;
    private double _height;

    // Публичный конструктор
    public Rectangle(double width, double height)
    {
        _width = width;
        _height = height;
    }

    // Публичный метод вычисления площади
    public double Area()
    {
        return _width * _height;
    }

    // Публичный метод вычисления периметра
    public double Perimeter()
    {
        return 2 * (_width + _height);
    }
}
```

Для проверки корректности реализации необходимо добавить файл с точкой входа в программу. По умолчанию в C# он называется `Program.cs`.

```csharp
// Также указываем, что работаем в том же пространстве имен
namespace Shapes;

// Класс-точка входа в программу
internal class Program
{
    // Метод-точка входа, аналог конструкции if __name__ == "__main__" в Python
    private static void Main(string[] args)
    {
        // Создание объекта прямоугольника
        Rectangle rect = new(4, 5);

        // В C# реализована строгая инкапсуляция, поэтому нельзя выполнить
        // присваивание вида:
        // rect._width = 10;
        // Такой код не скомпилируется.

        // Вывод результатов вычислений на экран (аналог функции print в Python)

        Console.WriteLine($"Периметр: {rect.Perimeter()}");

```

Теперь рассмотрим реализацию инкапсуляции в языке Go в файле `shapes.go`.

```go
// Обозначение пакета. Пакеты в Go во многом аналогичны пространствам имен в C#.
package shapes

// В Go нет классов, вместо них используются структуры, как в C и C++.
type Rectangle struct {
    // Приватные поля в Go обозначаются именами, начинающимися с маленькой буквы
    width  float64
    height float64
}

// В Go у структур нет конструктора в привычном ООП-смысле.
// Обычно принято создавать функцию вида NewИмяСтруктуры,
// которая возвращает указатель на созданную структуру.
func NewRectangle(width, height float64) *Rectangle {
    return &Rectangle{
        width:  width,
        height: height,
    }
}

// Публичные методы в Go задаются именами, начинающимися с заглавной буквы.
func (r *Rectangle) Area() float64 {
    return r.width * r.height
}

func (r *Rectangle) Perimeter() float64 {
    return 2 * (r.width + r.height)
}
```

Точку входа в программу на Go опишем в файле `main.go`.

```go
// Точки входа в программу располагаются в пакете с именем main
package main

// Импортируем пакет для работы с форматированным выводом и наш пакет с формами
import (
    "fmt"
    "shapes/shapes"
)

// Функция-точка входа
func main() {
    // Создаем наш прямоугольник
    rect := shapes.NewRectangle(4, 5)

    // Прямое присваивание rect.width = 10 недопустимо, программа не
    // скомпилируется, так как поле width является приватным.

    // Вывод результатов на экран
    fmt.Printf("Площадь: %.0f\n", rect.Area())
    fmt.Printf("Периметр: %.0f\n", rect.Perimeter())
}
```

Вернемся к Python и создадим файл с точкой входа `main.py`.

```python
# Импортируем класс Rectangle из модуля shapes.py
from shapes import Rectangle

# Функция-точка входа

def main():
    rect = Rectangle(4, 5)
    print(f"Площадь до: {rect.area()}")
    print(f"Периметр до: {rect.perimeter()}")

    # Следующая конструкция возможна, поскольку в Python нет настоящей
    # инкапсуляции на уровне языка, а действует лишь соглашение между
    # разработчиками.
    rect._height = 16
    print(f"Площадь после: {rect.area()}")
    print(f"Периметр после: {rect.perimeter()}")


if __name__ == "__main__":
    main()
```

Данный пример демонстрирует, что в Python защита внутренних полей носит рекомендательный характер: по соглашению считается, что атрибуты с одним подчеркиванием являются «приватными», однако язык не предотвращает прямого доступа к ним.

## Наследование и полиморфизм

Рассмотрим, как реализуются наследование и полиморфизм в разных языках программирования, и затем вернемся к Python.

Начнем с модификации файла `shapes.py` для Python.

```python
# Импортируем модуль с математическими функциями
import math

# Создадим общий класс форм, определив в нем общие методы и поля.
# Реализации методов в базовом классе не будет — это будет абстрактный класс.
class Shape:
    # Одно "приватное" поле — цвет
    def __init__(self, color):
        self._color = color

    # Метод площади (абстрактная заглушка)
    def area(self):
        pass

    # Метод периметра (абстрактная заглушка)
    def perimeter(self):
        pass

    # Геттер цвета
    def get_color(self):
        return self._color

    # Сеттер цвета
    def set_color(self, color):
        self._color = color


# Класс прямоугольника, наследующийся от Shape
class Rectangle(Shape):
    def __init__(self, width, height, color):
        super().__init__(color)
        self._width = width
        self._height = height

    def area(self):
        return self._width * self._height

    def perimeter(self):
        return 2 * (self._width + self._height)


# Класс круга, наследующийся от Shape
class Circle(Shape):
    def __init__(self, radius, color):
        super().__init__(color)
        self._radius = radius

    def area(self):
        return math.pi * self._radius ** 2

    def perimeter(self):
        return 2 * math.pi * self._radius
```

Перепишем аналогичную модель на C# в файле `shapes.cs`.

```csharp
namespace Shapes;

// Создаем абстрактный класс формы
abstract class Shape
{
    // Приватное поле цвет
    private string color;

    // Конструктор объявлен как protected, а не private, поскольку в C# гибкая
    // система уровней доступа. Приватные поля и методы доступны только внутри
    // самого класса. Чтобы сделать поле доступным для потомков, его либо
    // объявляют protected, либо предоставляют защищенный конструктор.
    protected Shape(string color)
    {
        this.color = color;
    }

    // Публичные геттер и сеттер для цвета, доступные всем потомкам
    public string GetColor()
    {
        return color;
    }

    public void SetColor(string color)
    {
        this.color = color;
    }

    // Абстрактные методы, которые обязан переопределить каждый потомок
    public abstract double Area();

    public abstract double Perimeter();
}

// Класс прямоугольника, наследующийся от Shape
class Rectangle : Shape
{
    private double width, height;

    public Rectangle(double width, double height, string color)
        : base(color)
    {
        this.width = width;
        this.height = height;
    }

    // Явное переопределение абстрактного метода
    public override double Area()
    {
        return width * height;
    }

    public override double Perimeter()
    {
        return 2 * (width + height);
    }
}

// Класс круга, наследующийся от Shape
class Circle : Shape
{
    private double radius;

    public Circle(double radius, string color) : base(color)
    {
        this.radius = radius;
    }

    public override double Area()
    {
        return Math.PI * radius * radius;
    }

    public override double Perimeter()
    {
        return 2 * Math.PI * radius;
    }
}
```

Продемонстрируем работу полиморфизма в файле `Program.cs`.

```csharp
using Shapes;

internal class Program
{
    private static void Main(string[] args)
    {
        // Создаем экземпляры прямоугольника и круга
        Rectangle rect = new Rectangle(10, 20, "Blue");
        Circle circle = new Circle(20, "Red");

        // Спокойно передаем прямоугольник и круг в метод, принимающий Shape,
        // так как оба класса являются потомками абстрактного класса Shape.
        PrintShapeData(rect);
        PrintShapeData(circle);

        Console.ReadKey();
    }

    // Метод, который выводит информацию о форме. В качестве параметра
    // используется абстрактный тип Shape. Таким образом, код не привязан
    // к конкретным реализациям, а опирается на общий интерфейс.
    private static void PrintShapeData(Shape shape)
    {
        Console.WriteLine($"Площадь: {shape.Area()}");
        Console.WriteLine($"Периметр: {shape.Perimeter()}");
        Console.WriteLine($"Цвет: {shape.GetColor()}");
    }
}
```

Идея полиморфизма состоит в том, что можно писать код, ориентируясь на некоторый «абстрактный» тип, не зная заранее, какой именно конкретный класс будет использоваться в дальнейшем. 
Достаточно знать, что объект реализует определенный набор методов и полей. Подобный подход часто называют «утинной типизацией»: если некий объект ведет себя как утка (имеет соответствующие методы), то его можно рассматривать как утку.

Рассмотрим теперь реализацию полиморфизма на языке Go.

```go
package shapes

import (
    "math"
)

// В Go нет наследования в классическом ООП-смысле. Полиморфизм реализуется
// через специальные типы — интерфейсы. В интерфейсе перечисляются методы,
// которые должна реализовать структура, чтобы удовлетворять этому интерфейсу.
// Это еще один пример утинной типизации.
type Shape interface {
    Area() float64
    Perimeter() float64
    GetColor() string
    SetColor(string)
}

// В Go отсутствует наследование, но используется композиция — способ создания
// сложных объектов путем включения в них других объектов как составных частей.
// Для этого создадим вспомогательную структуру, которую будем встраивать в
// другие структуры.
type shape struct {
    color string
}

func (s *shape) GetColor() string {
    return s.color
}

func (s *shape) SetColor(c string) {
    s.color = c
}

// Структура прямоугольника, использующая композицию

type Rectangle struct {
    // Встраивание структуры shape — техническая реализация композиции
    shape
    width  float64
    height float64
}

func NewRectangle(width, height float64, color string) *Rectangle {
    return &Rectangle{
        // Встраивание по сути аналогично наличию поля типа shape,
        // но нам не нужно явно указывать имя поля при обращении.
        shape:  shape{color: color},
        width:  width,
        height: height,
    }
}

func (r *Rectangle) Area() float64 {
    return r.width * r.height
}

func (r *Rectangle) Perimeter() float64 {
    return 2 * (r.width + r.height)
}

// Структура круга, также использующая композицию

type Circle struct {
    shape
    radius float64
}

func NewCircle(radius float64, color string) *Circle {
    return &Circle{
        shape:  shape{color: color},
        radius: radius,
    }
}

func (c *Circle) Area() float64 {
    return math.Pi * c.radius * c.radius
}

func (c *Circle) Perimeter() float64 {
    return 2 * math.Pi * c.radius
}
```

Демонстрация полиморфизма в модуле `main.go`.

```go
package main

import (
    "fmt"
    "shapes/shapes"
)

// Функция принимает параметр интерфейсного типа Shape.
// В нее можно передать любой тип, реализующий методы интерфейса Shape.
func PrintShapeData(shape shapes.Shape) {
    fmt.Printf("Цвет: %s\n", shape.GetColor())
    fmt.Printf("Площадь: %.2f\n", shape.Area())
    fmt.Printf("Периметр: %.2f\n", shape.Perimeter())
    fmt.Println("---")
}

func main() {
    rect := shapes.NewRectangle(10, 20, "Blue")
    circle := shapes.NewCircle(20, "Red")

    PrintShapeData(rect)
    PrintShapeData(circle)
}
```

Можно заметить, что реализация полиморфизма через интерфейсы и композицию в Go часто оказывается более гибкой, чем через наследование в классическом ООП. В языке Python, как языке с динамической типизацией, в качестве параметра функции можно передать практически любой объект. Ограничения в основном накладываются соглашениями между разработчиками и архитектурой программного обеспечения.

Рассмотрим пример на Python.

```python
from shapes import Rectangle, Circle, Shape


# Как видно из кода, язык не накладывает жестких ограничений на тип параметра.
# Главное — чтобы объект поддерживал необходимые методы.
def print_shape_data(shape):
    print(f"Цвет: {shape.get_color()}")
    print(f"Площадь: {shape.area():.2f}")
    print(f"Периметр: {shape.perimeter():.2f}")
    print("---")


def main():
    rect = Rectangle(10, 20, "Blue")
    circle = Circle(20, "Red")

    # Также можно явно создавать экземпляр абстрактного класса
    shape = Shape("Green")

    print_shape_data(rect)
    print_shape_data(circle)


if __name__ == "__main__":
    main()
```

## Более строгий ОО-подход в Python

Возникает естественный вопрос: можно ли сделать объектно-ориентированную модель в Python более «классической» и строгой? В определенной степени да — за счет использования дополнительных модулей и аннотаций типов (type hints).

Проверку типов на уровне самого языка включить нельзя, однако на практике широко применяются внешние утилиты, анализирующие код статически. Рассмотрим пример переписывания модулей с использованием абстрактных базовых классов и аннотаций типов.

```python
# Подключаем модули для реализации абстрактного класса и математических функций
from abc import ABC, abstractmethod
from math import pi


# Если класс наследуется от ABC, то он считается абстрактным.
# Благодаря этому напрямую создать экземпляр такого класса нельзя.
class Shape(ABC):
    def __init__(self, color: str):
        self._color = color

    def get_color(self) -> str:
        return self._color

    def set_color(self, color: str) -> None:
        self._color = color

    # Декоратор @abstractmethod используется для проверки того,
    # что классы-потомки реализуют соответствующий метод.
    @abstractmethod
    def area(self) -> float:
        pass

    @abstractmethod
    def perimeter(self) -> float:
        pass


class Rectangle(Shape):
    def __init__(self, width: float, height: float, color: str):
        super().__init__(color)
        self._width = width
        self._height = height

    def area(self) -> float:
        return self._width * self._height

    def perimeter(self) -> float:
        return 2 * (self._width + self._height)


class Circle(Shape):
    def __init__(self, radius: float, color: str):
        super().__init__(color)
        self._radius = radius

    def area(self) -> float:
        return pi * self._radius ** 2

    def perimeter(self) -> float:
        return 2 * pi * self._radius
```

Как видно, повсеместно были добавлены аннотации типов.

```python
from shapes import Rectangle, Circle, Shape


def print_shape_data(shape: Shape) -> None:
    print(f"Цвет: {shape.get_color()}")
    print(f"Площадь: {shape.area():.2f}")
    print(f"Периметр: {shape.perimeter():.2f}")
    print("---")


def main():
    rect = Rectangle(10, 20, "Blue")
    circle = Circle(20, "Red")

    print_shape_data(rect)
    print_shape_data(circle)


if __name__ == "__main__":
    main()
```

Существует несколько способов приближать поведение Python к строгой типизации. Помимо абстрактных классов можно подключать сторонние библиотеки, например `typeguard` или `pydantic`, чтобы проверять типы во время выполнения. Однако это достаточно ресурсоемкие решения.

На практике чаще используют статические анализаторы (линтеры), как интегрированные в IDE, так и внешние инструменты, например `mypy`.

```bash
pip install mypy
mypy shapes.py main.py
```

Таким образом, ключевая идея полиморфизма и ООП состоит в том, что можно писать код, не задумываясь о том, с какой конкретной сущностью предстоит работать. Например, возможно реализовать логику, способную работать с любыми базами данных, в то время как детали взаимодействия с конкретной СУБД будут реализованы другим разработчиком в виде конкретного класса или модуля.

## Функциональное программирование

Функциональное программирование (ФП) — это парадигма программирования, основанная на идеях лямбда-исчисления. Одно из основных ограничений, накладываемых данной парадигмой, заключается в ограничении или полном исключении изменяемых присваиваний.

С точки зрения функционального подхода переменные, как правило, рассматриваются как неизменяемые. Возникает вопрос: зачем это нужно? Прежде всего для многопоточных приложений. Если существует изменяемая переменная, к которой одновременно обращаются несколько потоков, между ними возможно состязание за доступ к данным. Такая ситуация называется гонкой данных (race condition).

Одним из способов избежать подобных ситуаций является отказ от изменения уже существующих данных и создание новых значений взамен. По такому принципу, в частности, организуется работа современных банковских баз данных: они хранят множество записей о транзакциях (изменениях состояния счета), а текущее состояние счета определяется не одной переменной, а совокупностью строк, на основе которых вычисляется итоговый баланс.

Для целей данной лекции функциональное программирование важно не в контексте асинхронности (этот вопрос в рамках курса подробно не рассматривается), а как иллюстрация иной парадигмы мышления при проектировании программ.

Рассмотрим небольшой пример кода на языке, ориентированном на функциональное программирование (Clojure).

```clojure
(println (take 25 (map (fn [x] (* x x)) (range))))
```

Тот же пример, но с более подробным форматированием:

```clojure
(println
    (take 25
        (map (fn [x] (* x x))
             (range))))
```

В данном языке вызов функции осуществляется путем записи имени функции и аргументов в круглых скобках. В примере функция `range` возвращает (потенциально) бесконечную последовательность. Результат передается в функцию `map`, которая применяет к каждому элементу анонимную функцию (также называемую лямбда-функцией). Функция `map` относится к функциям высшего порядка, так как принимает другую функцию в качестве аргумента. Функция, передаваемая в качестве аргумента, называется функцией первого порядка.

Перейдем теперь к примерам функциональных элементов в Python.

Во-первых, в Python ряд типов данных является неизменяемым (immutable): это, например, целые числа, строки, кортежи и некоторые другие базовые типы. Во-вторых, в языке существуют анонимные функции (lambda-выражения).

```python
# Присваиваем переменной add анонимную функцию, складывающую два числа
add = lambda x, y: x + y

# Аналогичный код можно записать, используя обычную (именованную) функцию

def add(x, y):
    return x + y

# Присваиваем переменной a функцию add. Функции в Python — объекты первого
# порядка, их можно передавать и присваивать переменным.
a = add

# Функция, вычисляющая квадрат числа (анонимная функция)
sqr = lambda x: x * x

# Список элементов, для которых требуется получить квадраты
l = [1, 2, 3, 4, 5]

# Чтобы получить квадраты элементов, можно не использовать явный цикл,
# а воспользоваться функцией высшего порядка map
l_sqr = map(sqr, l)
```

Практическая значимость этих конструкций состоит в том, что они часто встречаются в реальном коде. Например, студенты нередко используют следующий фрагмент:

```python
l = map(int, input().split(' '))
```

На вопрос, что именно делает данная конструкция, не всегда находится четкий ответ. Разберем ее по шагам.

Функция `input()` возвращает строку, введенную пользователем в консоли. Метод строки `split` (пример использования ООП: метод объекта типа `str`) возвращает список подстрок, разделенных указанным разделителем. Функция `int` в данном контексте выступает как обычная функция с одним аргументом. Функция высшего порядка `map` применяет ее к каждому элементу списка, полученного из `split`.

В результате формируется объект, который можно трактовать как последовательность целых чисел. При необходимости эту последовательность можно привести к списку с помощью `list(l)`.

## Заключение

Python является мультипарадигменным языком программирования. Он поддерживает как объектно-ориентированный, так и функциональный стиль, а также позволяет комбинировать элементы разных подходов в одном проекте. 

Когда говорится, что "все в Python является объектом", имеется в виду, что где-то в исходных кодах интерпретатора определены классы, соответствующие базовым типам (например, `str`), которые содержат определенные поля и методы. Благодаря этому можно использовать вызовы методов непосредственно у литералов строкового типа, например:

```python
l = 'Привет мир'.split()  # создаст список ['Привет', 'Мир']
```

Понимание принципов ООП (инкапсуляция, наследование, полиморфизм) и функционального программирования (неизменяемость, функции высшего порядка, лямбда-выражения) позволяет осознанно выбирать архитектурные решения и стиль кода в зависимости от задач и особенностей языка.
