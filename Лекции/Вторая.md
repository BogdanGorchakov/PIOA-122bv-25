## Функции в Python

### Понятие функции

Функция — это **именованный блок кода**, который можно вызывать многократно, передавая ему аргументы и получая результат. По сути, функция — это подпрограмма внутри программы.

Функции используются для решения следующих задач:

- Снижение количества дублирующегося кода, а следовательно, уменьшение числа ошибок.
- Декомпозиция программы на логические части, что повышает читаемость и сопровождаемость кода.

Пример уже знакомой вам функции — `print`. Это функция, предназначенная для вывода сообщения в стандартный поток вывода (как правило, консоль). Её внутренний алгоритм достаточно сложен, но его можно описать следующим образом:

- Преобразование переданных аргументов в строковое представление.
- Объединение этих строк через заданный разделитель.
- Добавление в конце строки окончания (по умолчанию `"\n"`).
- Вывод итоговой строки в указанный поток (по умолчанию консоль, но может быть и файл и т.п.).

Если бы в языке не было функций, подобный алгоритм пришлось бы встраивать в каждое место, где требуется вывод, дублируя один и тот же код. Это привело бы к:

- появлению большого количества однотипных фрагментов;
- ухудшению читаемости программы;
- значительному росту вероятности ошибок, особенно при изменении логики вывода.

Как говорил создатель Python Гвидо ван Россум: «Код читают гораздо чаще, чем пишут». Поэтому минимизация дублирования и повышение читаемости являются важнейшими целями.

### Определение и вызов функции в Python

Для определения функции в Python используется ключевое слово `def` (сокращение от английского **define** — «определять»). Рассмотрим простейшую функцию, которая выводит на экран слово `"Hello"`:

```python
# Определяем функцию hello
def hello():
    # Тело функции выделяется отступами (в Python принято использовать 4 пробела)
    print('Hello')

# Вызов функции осуществляется по имени с последующими скобками
hello()
```

Данная функция при вызове просто выводит на экран строку `"Hello"`.

### Параметры функции: позиционные, значения по умолчанию

Рассмотрим пример функции с параметром:

```python
# Параметры функции записываются в круглых скобках после имени функции.
# Такие параметры называются формальными параметрами или аргументами.
def hello(name):
    # После объявления параметра его можно использовать как обычную переменную
    print('Hello', name)

# Передача аргумента при вызове функции:
# передаваемое значение называется фактическим параметром.
hello('world')

# Если вызвать функцию без обязательного параметра, интерпретатор выдаст исключение:
hello()  # так делать нельзя
```

Отсутствие значения можно компенсировать использованием **значения по умолчанию**:

```python
# Параметру задаётся значение по умолчанию с помощью оператора =
def hello(name='world'):
    print('Hello', name)

hello()        # в этом случае будет использовано значение по умолчанию: Hello world
hello('test')  # будет выведено: Hello test
```


### Позиционная передача параметров

По умолчанию параметры в функциях передаются по позиции:

```python
def hello(firstName, secondName):
    print('Hello', firstName, secondName)

hello('Вася', 'Петров')
# Здесь firstName = 'Вася', secondName = 'Петров'
# Вывод: Hello Вася Петров

hello('Петров', 'Вася')
# Здесь firstName = 'Петров', secondName = 'Вася'
# Вывод: Hello Петров Вася
```


### Возврат значений. Оператор `return`

Все приведённые выше примеры не возвращали явного значения. В некоторых языках программирования (например, Pascal) подобные конструкции называли бы процедурами.

В Python любая функция **всегда** возвращает значение: если оператор `return` не указан явно, возвращается специальный объект `None` (литерал «пустоты»).

Для возврата результата используется оператор `return`. Рассмотрим функцию поиска максимума из двух чисел:

```python
# Определим функцию max2, принимающую два значения
def max2(a, b):
    if a > b:
        # После оператора return дальнейшее выполнение функции прекращается.
        # Этот приём иногда называют "ранний возврат".
        # Он позволяет не писать блок else, сокращая объём кода и повышая читаемость.
        return a
    return b

m = max2(1, 2)
print(m)  # будет выведено 2
```

Ту же функцию можно переписать с использованием **тернарного оператора**:

```python
def max2(a, b):
    # Читается: a, если a > b, иначе b
    m = a if a > b else b
    return m
```


### Вложенные вызовы функций

Результат работы одной функции можно передать в другую. Например, реализуем поиск максимального из трёх чисел, используя функцию `max2`:

```python
def max2(a, b):
    return a if a > b else b

def max3(a, b, c):
    # Сначала вычисляется max2(b, c),
    # затем результат передаётся в max2(a, результат)
    return max2(a, max2(b, c))

m = max3(1, 9, 2)
print(m)  # будет 9
```

Все рассмотренные выше идеи характерны не только для Python, но и для многих других языков программирования.

### Именованные аргументы и произвольное число параметров

Теперь рассмотрим особенности Python, связанные с тем, **как** функции принимают аргументы.

Во-первых, Python поддерживает **именованную передачу аргументов**:

```python
def hello(firstName, secondName):
    print('Hello', firstName, secondName)

hello(secondName='Вася', firstName='Петров')
# Здесь firstName = 'Петров', secondName = 'Вася'
# Вывод: Hello Петров Вася
```

Во-вторых, в Python реализована возможность передавать **произвольное количество позиционных параметров** с помощью `*` и **произвольное количество именованных параметров** с помощью `**`.

Рассмотрим функцию `print`. Её сигнатура (упрощённо) выглядит так:

```python
def print(*values, sep=" ", end="\n", file=None, flush=False):
    ...
```

Комментарии к параметрам:

- `*values` — любой набор позиционных аргументов, которые будут выведены;
- `sep` — строка-разделитель между значениями (по умолчанию пробел `" "`);
- `end` — строка, добавляемая в конце вывода (по умолчанию `"\n"`);
- `file` — объект, в который производится вывод (по умолчанию стандартный поток вывода, т.е. консоль);
- `flush` — флаг, указывающий на необходимость принудительной очистки буфера.

Разберёмся с `*values`. Звёздочка перед параметром означает, что функция может принимать **произвольное количество позиционных значений**. Внутри функции эти значения доступны как **кортеж** `values` (кортеж — отдельная тема, будет разобрана позже).

Из-за того, что первый параметр `values` принимает произвольное число аргументов, при вызове функции не всегда очевидно, как изменить, например, `sep`. Для этого и существуют **именованные аргументы**:

```python
print("Hello", "World", sep="-")
# Вывод: Hello-World
```

Аналогично можно определять функции, принимающие произвольное число именованных аргументов:

```python
def hello(**data):
    # В этом случае data — словарь, где ключи — имена параметров,
    # а значения — переданные аргументы.
    print('Hello', data["first_name"], data["second_name"], 'from', data["city"])

hello(first_name="Вася", second_name="Петров", city="Москва")
# Вывод: Hello Вася Петров from Москва
```

## Стек вызовов

После вызова функции выполнение кода в месте вызова приостанавливается до тех пор, пока функция не завершит работу. Схематично это можно описать так:

- выполняется код функции `main`;
- в какой-то момент вызывается функция `hello`;
- управление передаётся функции `hello`;
- после завершения `hello` выполнение продолжается в `main` после точки вызова.

То есть, функции в обычном режиме работают **синхронно**. Параллельное/одновременное выполнение двух функций относится уже к тематике асинхронного программирования и выходит за рамки данной лекции.

Возникает вопрос: как интерпретатор «помнит», куда нужно вернуться после завершения функции? Для этого используется структура данных **стек вызовов**.

### Понятие стека

Стек — это структура данных с семантикой LIFO (Last In — First Out, «последним пришёл — первым вышел»).

Пример реальной структуры с подобным поведением — ханойская башня: можно снять только то кольцо, которое сверху (то есть было помещено последним). Аналогично, элементы стека снимаются в обратном порядке относительно добавления.

**Стек вызовов** работает по тому же принципу: при вызове функции информация о месте вызова помещается в стек, а при завершении функции она извлекается, и управление возвращается туда, откуда был произведён вызов.

Рассмотрим пример:

```python
def a():
    # 2. Внутри функции a выполнение доходит до вызова функции b.
    # В стек записывается информация о месте вызова (функция a, номер строки).
    b()

def b():
    # 3. Внутри функции b выполнение доходит до вызова функции c.
    # В стек записывается информация о месте вызова (функция b, номер строки).
    c()

def c():
    # 4. По завершении функции c управление возвращается по стеку назад.
    pass

# 1. Выполнение программы доходит до вызова функции a.
# В стек записывается информация о месте вызова (main, номер строки).
a()
```

Схематически содержимое стека в некоторый момент времени можно представить так:


| Функция | Информация стека | Номер кадра в стеке |
| :-- | :-- | :-- |
| c | b, строка 1 | 2 |
| b | a, строка 1 | 1 |
| a | main, строка 1 | 0 |

Если, например, в функции `c` произойдёт ошибка, интерпретатор сбросит весь стек и выведет трассировку (traceback), показывая, в каких местах и в каком порядке были сделаны вызовы, приведшие к ошибке.

Если функция будет вызывать другую функцию (или саму себя) бесконечно, произойдёт **переполнение стека** (stack overflow). Это типичная проблема при неправильной реализации рекурсивных алгоритмов. Важно понимать, что стек занимает память, и его размер ограничен.

На этом обзор функций можно завершить и перейти к рассмотрению парадигм программирования.

## Парадигмы программирования

В современном программировании выделяют три основные парадигмы:

- структурное программирование;
- объектно-ориентированное программирование;
- функциональное программирование.

**Парадигма программирования** — это способ организации программ, не зависящий от конкретного языка. Парадигма определяет, какие структуры и приёмы следует использовать и в каких ситуациях.

### Структурное программирование

Потребность в парадигмах возникла из-за сложности программирования как вида деятельности: крупные программы создавать и сопровождать трудно, и люди часто ошибаются. Это привело к идее **ограничения свободы программиста** и внедрения дисциплинирующих правил.

Первой широко принятой парадигмой стало **структурное программирование**, которое накладывает ограничения на прямую передачу управления.

#### Оператор `goto` и его проблемы

В ранних языках программирования существовал оператор безусловного перехода `goto`, позволявший «прыгать» к произвольному месту программы. Постоянное использование `goto` приводило к появлению так называемого «спагетти-кода» — сложного, запутанного и трудно сопровождаемого.

В 1950–1960‑е годы было строго доказано, что любую программу можно написать, используя всего три базовые управляющие структуры:

- последовательность;
- выбор (ветвление, например, `if`);
- итерацию (циклы).

В современных языках программирования оператор `goto`, если и существует, то используется крайне редко и с серьёзными ограничениями. В результате можно сказать, что практически все разработчики сознательно или неосознанно пишут в парадигме структурного программирования.

#### Функциональная декомпозиция

Структурное программирование позволило сформировать подход **функциональной декомпозиции**: любую программу можно разложить на набор функций верхнего уровня, которые, в свою очередь, декомпозируются на более простые функции.

Именно это мы уже наблюдали при использовании функции `print`: с точки зрения основной программы вывод выглядит как единое действие, хотя внутри скрыт достаточно сложный алгоритм.

Благодаря структурному программированию:

- программа разбивается на множество небольших, легко проверяемых функций;
- упрощается тестирование кода (каждую функцию можно проверять отдельно).

**Итог**: структурное программирование опирается на использование условий, циклов, функций и модулей.

## Объектно-ориентированное программирование (ООП)

Второй по значимости парадигмой является **объектно-ориентированное программирование**.

Для начального объяснения ООП нередко используют упрощённую формулировку: «ООП — это способ отражения (моделирования) реального мира в программном коде». Это определение не вполне корректно, однако, опираясь на него, можно сделать первые шаги в понимании ООП.

Во многих языках программирования существует сущность **класс**, к числу таких языков относится и Python.

### Классы и объекты

Рассмотрим базовый пример:

```python
# Определим класс Person, представляющий информацию о человеке.
# Класс можно рассматривать как "чертёж" будущих объектов.
# Объекты (экземпляры класса) — это конкретные элементы, созданные по этому чертежу.

class Person:

    # В Python у классов есть конструктор.
    # Конструктор (__init__) вызывается при создании экземпляра класса.
    # В конструкторе обычно определяются поля (атрибуты) объекта.
    # Поле (или атрибут) — это переменная, привязанная к объекту.
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # Определим метод talk.
    # Метод — это функция, связанная с объектом (экземпляром класса).
    def talk(self):
        print('Я', self.name, 'мне', self.age)

# Создание объекта осуществляется путём вызова имени класса как функции.
# По сути это приводит к вызову конструктора __init__, которому передаются аргументы.
p = Person('Вася', 30)

# Обращение к полям выполняется через оператор точки
print(p.name)  # Вася

# Значения полей можно изменять
p.age = 44
print(p.age)  # 44

# Аналогично обращаемся к методам через точку
p.talk()  # Я Вася мне 44
```

Таким образом:

- **класс** задаёт структуру и поведение объектов (какие поля и методы у них есть);
- **объект** — конкретный экземпляр с определёнными значениями полей.


### Классические «три кита» ООП

Обычно ООП описывают через три ключевых понятия:

- инкапсуляция;
- наследование;
- полиморфизм.

---

### Инкапсуляция

Инкапсуляция — это идея **скрытия внутренней реализации** объекта от внешнего кода. Пользователь объекта должен иметь доступ только к «публичному» интерфейсу (методы и поля, предназначенные для использования), при этом детали реализации могут быть изменены без влияния на внешний код.

Рассмотрим пример:

```python
class Person:

    # Добавим скрытые поля: год рождения и возраст.
    # В Python принято считать, что имена, начинающиеся с "_", являются "приватными".
    def __init__(self, name, year_of_birth):
        self.name = name
        self._year_of_birth = year_of_birth
        # Предположим, что существует функция get_current_year(),
        # которая возвращает текущий год (в реальности можно использовать модуль datetime).
        self._age = get_current_year() - year_of_birth

    # Если поле скрыто, то для доступа к нему обычно создаются методы-геттеры.
    def get_year_of_birth(self):
        return self._year_of_birth

    def get_age(self):
        return self._age

    # Для изменения скрытого поля используются методы-сеттеры.
    def set_year_of_birth(self, year_of_birth):
        self._year_of_birth = year_of_birth
        self._age = get_current_year() - year_of_birth

    def talk(self):
        print('Я', self.name, 'мне', self._age)
```

Использование:

```python
p = Person('Вася', 2000)
p.talk()                 # Я Вася мне 26 (на момент 2026 года)
print(p.get_year_of_birth())  # 2000

# Можно написать и так:
p._age = 25
```

С технической точки зрения **ничто не мешает** обратиться к полю `_age` напрямую: в Python нет жёсткого механизма инкапсуляции в стиле некоторых других языков. Существует **соглашение**: если имя поля или метода начинается с `_`, то разработчик как бы заявляет: «Не используйте это напрямую, это внутреннее». Ожидается, что программисты будут следовать этому соглашению.

### Наследование

Наследование позволяет создавать новые классы на основе существующих, **перенося** общую часть функциональности в базовый (родительский) класс.

Предположим, у нас есть класс `Person`. Можно создать класс `Worker`, который расширяет поведение `Person`, добавляя новые методы и поля.

Сначала покажем «наивную» реализацию без наследования:

```python
class Worker:

    def __init__(self, name, year_of_birth):
        self.name = name
        self._year_of_birth = year_of_birth
        self._age = get_current_year() - year_of_birth

    def get_year_of_birth(self):
        return self._year_of_birth

    def get_age(self):
        return self._age

    def set_year_of_birth(self, year_of_birth):
        self._year_of_birth = year_of_birth
        self._age = get_current_year() - year_of_birth

    def talk(self):
        print('Я', self.name, 'мне', self._age)

    def work(self):
        print('Я работаю')
```

Здесь видно, что большая часть кода повторяет функциональность `Person`. Применим наследование:

```python
class Person:
    def __init__(self, name, year_of_birth):
        self.name = name
        self._year_of_birth = year_of_birth
        self._age = get_current_year() - year_of_birth

    def get_year_of_birth(self):
        return self._year_of_birth

    def get_age(self):
        return self._age

    def set_year_of_birth(self, year_of_birth):
        self._year_of_birth = year_of_birth
        self._age = get_current_year() - year_of_birth

    def talk(self):
        print('Я', self.name, 'мне', self._age)


# Наследуемся от Person
class Worker(Person):

    # Переопределяем конструктор, добавляя новый параметр experience
    def __init__(self, name, year_of_birth, experience):
        # Вызов конструктора родительского класса
        super().__init__(name, year_of_birth)
        self.experience = experience

    def work(self):
        print('Я работаю')
```

Использование:

```python
w = Worker('Вася', 2000, 5)

w.talk()         # Я Вася мне 26 (условно)
w.work()         # Я работаю
print(w.experience)  # 5
```

Таким образом:

- общая логика вынесена в родительский класс `Person`;
- в `Worker` добавлены или переопределены методы и поля, специфичные для работника.
