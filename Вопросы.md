# Вопросы по лекциям

## Теоретические вопросы

1. Объясните, что такое однопроходный алгоритм. Почему ограничение на один проход по данным влияет на выбор переменных состояния и на то, какие задачи можно решить эффективно?
2. Чем интерпретируемое выполнение Python отличается от компиляции в машинный код? Какие преимущества и ограничения это даёт при разработке и отладке программ?
3. Объясните различие между динамической и статической типизацией, а также между сильной и слабой типизацией. Почему сильная типизация помогает раньше обнаруживать ошибки контракта между частями программы?
4. Как стек вызовов помогает интерпретатору возвращаться из вложенных вызовов функций? Почему бесконечная рекурсия приводит к переполнению стека?
5. Зачем функции нужны параметры по умолчанию, именованные аргументы, `*args` и `**kwargs`? В каких случаях их использование делает код понятнее, а в каких может запутать читателя?
6. Объясните смысл инкапсуляции на примере класса. Почему прямой доступ к полям объекта может быть опасен для сопровождения программы?
7. В чём различие между наследованием и полиморфизмом? Почему общий интерфейс, например для `MemoryDatabase` и `FileDatabase`, позволяет заменить одну реализацию другой без изменения пользовательского кода?
8. Сравните объектно-ориентированный и функциональный подходы на простом примере обработки данных. Какие задачи удобнее выражать через объекты, а какие через функции высшего порядка?
9. Что показывает асимптотическая сложность алгоритма и почему при её оценке обычно отбрасывают константы? Приведите пример, когда `O(log n)` принципиально лучше `O(n)`.
10. Почему вставка или удаление элемента в начале массива обычно имеет сложность `O(n)`, а доступ по индексу - `O(1)`? Как это связано с расположением данных в памяти?
11. Сравните сортировку пузырьком, выбором, вставками, быструю сортировку, сортировку слиянием и сортировку подсчётом. Для каждой группы объясните, когда алгоритм может быть уместен и какие у него ограничения.
12. Зачем в TDD сначала пишут тест, который падает? Как такой подход помогает уточнить интерфейс структуры данных, например стека, очереди или таблицы базы данных?
13. Сравните очередь на обычном массиве, кольцевом буфере и связанном списке. Почему операция удаления из начала списка Python плохо подходит для эффективной очереди?
14. Почему хеш-таблицы в среднем дают быстрый доступ по ключу, но не гарантируют `O(1)` в худшем случае? Объясните роль коллизий, коэффициента заполнения и рехеширования.
15. Сравните DFS, BFS, топологическую сортировку и алгоритм Дейкстры. Для каких задач подходит каждый из этих алгоритмов и какие ограничения важно помнить, например наличие циклов или отрицательных весов?

## Практические вопросы

1. Дан код:

   ```python
   def max2(a, b):
       return a if a > b else b

   def max3(a, b, c):
       return max2(max2(a, b), c)

   print(max3(4, 9, 7))
   ```

   Что будет выведено? Покажите порядок вызовов функций и объясните, какие значения возвращаются на каждом шаге.

2. Найдите ошибку в коде и перепишите функцию так, чтобы она возвращала сумму чисел от `0` до `n` включительно:

   ```python
   def sum_to_n(n):
       total = 0
       i = 0
       while i <= n:
           total += i
       return total
   ```

3. Напишите функцию `count_positive(numbers)`, которая за один проход по списку считает количество положительных чисел. Объясните, какие данные функция должна хранить во время прохода.

4. Дан код:

   ```python
   class Person:
       def __init__(self, name, year):
           self.name = name
           self._year = year

       def get_age(self, current_year):
           return current_year - self._year

   p = Person("Анна", 2005)
   print(p.get_age(2026))
   ```

   Что будет выведено? Почему вычисление возраста лучше оформить методом, а не хранить возраст как неизменяемое поле?

5. Допишите метод `area` в классе `Rectangle` и объясните, почему этот метод относится к поведению объекта:

   ```python
   class Rectangle:
       def __init__(self, width, height):
           self.width = width
           self.height = height

       # допишите метод area
   ```

6. Для класса `Vector` допишите метод `__add__`, чтобы выражение `Vector(1, 2) + Vector(3, 4)` возвращало новый вектор с координатами `(4, 6)`:

   ```python
   class Vector:
       def __init__(self, x, y):
           self.x = x
           self.y = y
   ```

7. Дан фрагмент сортировки вставками:

   ```python
   arr = [5, 2, 4]
   key = arr[1]
   j = 0

   while j >= 0 and arr[j] > key:
       arr[j + 1] = arr[j]
       j -= 1

   arr[j + 1] = key
   ```

   Чему будет равен `arr` после выполнения фрагмента? Поясните, зачем нужен сдвиг элементов вправо.

8. Выполните вручную первые три шага бинарного поиска числа `13` в массиве `[1, 3, 5, 7, 9, 11, 13, 15]`. Укажите значения `left`, `right` и `mid` на каждом шаге.

9. Для массива `[4, 1, 3, 1, 0]`, где все значения лежат от `0` до `4`, постройте массив подсчётов для сортировки подсчётом и восстановите отсортированный результат.

10. Есть стек, над которым выполняют операции:

    ```text
    push(10)
    push(20)
    pop()
    push(30)
    peek()
    pop()
    ```

    Какие значения вернут операции `pop` и `peek`? Что останется в стеке после всех операций?

11. Найдите логическую ошибку в очереди на списке Python:

    ```python
    class Queue:
        def __init__(self):
            self.items = []

        def enqueue(self, value):
            self.items.append(value)

        def dequeue(self):
            return self.items.pop()
    ```

    Почему такая реализация нарушает принцип FIFO и как исправить метод `dequeue`?

12. Дан код:

    ```python
    text = "ок"
    data = text.encode("utf-8")
    print(len(text))
    print(len(data))
    ```

    Какие два числа будут выведены? Объясните разницу между количеством символов строки и количеством байтов в UTF-8. Почему при чтении текстового файла важно явно понимать его кодировку?

13. Нужно хранить записи студентов вида `id -> имя` и быстро находить имя по `id`. Выберите структуру данных из изученных в лекциях и обоснуйте выбор. Как изменится ответ, если дополнительно нужно сохранять эти записи в файл: какой формат из CSV, JSON и XML вы бы выбрали для простой табличной структуры?

14. Для хеш-таблицы из `5` корзин используется правило `bucket = key % 5`. В какие корзины попадут ключи `12`, `17`, `20`, `25`? Где возникнут коллизии и как их можно обработать методом цепочек?

15. Дан ориентированный граф в виде списков смежности:

    ```python
        [1, 2],
        [3],
        [3],
        []
    ]
    ```

    Выполните BFS из вершины `0`: укажите порядок посещения вершин и расстояние от `0` до каждой вершины. Почему BFS даёт кратчайшие расстояния именно в невзвешенном графе, а для взвешенного графа с неотрицательными весами обычно нужен алгоритм Дейкстры?
