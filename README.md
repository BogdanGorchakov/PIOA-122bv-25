# Лабораторные работы №2 и №4: Система управления данными (СУБД)

## 1. Структура проекта (src-layout)
Проект организован в виде единого пакета со следующей архитектурой:
- `src/` — исходный код программы
  - `db/backend/` — логика базы данных
    - `database.py` — модель сущности Game (ЛР №2)
    - `memory.py` — СУБД в оперативной памяти (InMemoryDatabase) (ЛР №2)
    - `file.py` — файловые СУБД (JsonDatabase, CsvDatabase) (ЛР №4)
    - `errors.py` — кастомные исключения и обработка ошибок (ЛР №4)
  - `tui.py` — текстовый пользовательский интерфейс (TUI) с поддержкой фильтрации
  - `__main__.py` — главная точка входа для запуска
- `tests/` — автоматизированные тесты (pytest)
  - `test_memory.py` — тесты СУБД в памяти (ЛР №3)
  - `test_file_database.py` — тесты файловых СУБД (ЛР №4)

## 2. Реализация СУБД и различия между версиями

### Лабораторная работа №2 (In-Memory СУБД):
Класс `InMemoryDatabase` реализует полноценную СУБД в оперативной памяти. Поддерживает операции:
- Добавление записей с валидацией полей (название, жанр, диапазон года, ID студии).
- Удаление записей по ID.
- **Выборка и фильтрация:** метод `get_games()` позволяет искать записи по одному или нескольким полям одновременно (по части названия без учета регистра, точному совпадению жанра или году).

### Лабораторная работа №4 (Файловая СУБД):
Классы `JsonDatabase` и `CsvDatabase` наследуют и расширяют логику In-Memory реализации, добавляя персистентность (сохранение на диске):
- При старте данные автоматически считываются из файлов (`.json` / `.csv`).
- При добавлении или удалении записей состояние базы мгновенно синхронизируется с файлом на диске, предотвращая потерю данных между запусками.

## 3. Инструкция по запуску и проверке

### Запуск программы (Интерфейс СУБД):
Для запуска программы выполните команду из корневой директории проекта:
```bash
python -m src
```
После запуска программа предложит выбрать режим работы: In-Memory (через файловые обертки) или чтение/запись на диск.

### Запуск автоматических тестов:
Для проверки покрытия кода и валидации всех бизнес-правил выполните:
```bash
pytest
```
