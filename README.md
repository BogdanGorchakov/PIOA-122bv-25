# Система управления базами данных (Игры)

### Структура проекта
* `src/models.py` — Модели данных (класс Game)
* `src/database.py` — Реализация In-Memory СУБД и файловых бэкендов (JSON, CSV)
* `src/interface.py` — Консольный интерфейс пользователя
* `src/main.py` — Главная точка входа для запуска приложения
* `tests/test_db.py` — Автоматизированные тесты проекта

### Различия между реализациями
* **In-Memory СУБД**: Хранит данные исключительно в оперативной памяти в процессе работы программы. Очищается при выходе.
* **Файловая СУБД (JSON / CSV)**: Расширяет In-Memory СУБД, автоматически синхронизируя любые изменения с диском. Данные сохраняются в виде структурированных таблиц с метаданными (заголовками колонок) между перезапусками приложения.

### Запуск приложения
```bash
python -m src.main
```

### Запуск тестов и проверка покрытия
```bash
python -m unittest discover -s tests
```
