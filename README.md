# Лабораторная работа №4

## 1. Структура проекта
Проект организован в виде пакета со следующей архитектурой:
- `src/` — исходный код программы
  - `db/backend/` — логика базы данных
    - `database.py` — модель сущности Game
    - `memory.py` — СУБД в оперативной памяти (InMemoryDatabase)
    - `file.py` — файловые СУБД (JsonDatabase, CsvDatabase)
    - `errors.py` — кастомные исключения и обработка ошибок
  - `tui.py` — текстовый пользовательский интерфейс
  - `__main__.py` — главная точка входа для запуска
- `tests/` — автоматизированные тесты (pytest)
  - `test_memory.py` — тесты СУБД в памяти
  - `test_file_database.py` — тесты файловых СУБД

## 2. Различия между In-Memory и файловой реализациями
- **In-Memory (InMemoryDatabase):** Хранит все записи в оперативной памяти (в списках). Обеспечивает максимальное быстродействие, но данные полностью теряются при завершении процесса.
- **Файловая реализация (JsonDatabase / CsvDatabase):** Наследует базовую логику In-Memory, но синхронизирует состояние с диском. При старте данные считываются из файлов (`.json` / `.csv`), а при любом добавлении или удалении — автоматически перезаписываются на диск, обеспечивая персистентность.

## 3. Инструкция по запуску и проверке

### Запуск программы:
Для запуска СУБД выполните команду из корневой директории:
```bash
python -m src
```

### Запуск автоматических тестов:
Для проверки работоспособности всех компонентов выполните:
```bash
pytest
```
